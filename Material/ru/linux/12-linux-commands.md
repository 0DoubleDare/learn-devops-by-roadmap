# Команды
Список часто используемых команд в Linux
## Навигация по файловой системе и по содержимому в файлах
### `cd <путь к директории>`
**cd** (Change Directory) - Меняет текущую директорию по заданному пути. Можно вводить как относительные пути (это может звучать как "Вот я нахожусь в конкретной директории, вот тебе продолжение пути поэтому проведи меня дальше по нему") так и абсолютные (начинается с `/` и означает "перейди в корень всей системы и только потом переходи по заданному пути" )
### `mkdir <имя директории>`
**mkdir** (Make Directory) - создает директорию с заданным именем. Можно указать в аргументе путь где нужно создать каталог и его имя. То есть `mkdir ~/configs/my-program-config` создаст директорию my-program-config в **/home/{ваше имя пользователя}/configs**
### `find`
Утилита для поиска файлов в директории
### `grep`
**grep** (Global Regular Expression Print) - Утилита для поиска и сортировки содержимого в файле

## Процессы
### `pgrep <имя запущенной программы>` 

**pgrep** (Program Grep) - требует название запущенной программы в качестве аргумента (команда поддерживает автодополнение) и выводит его PID 
### `ps`

**ps** (Process Status) - показывает список процессов запущенные от имени текущего пользователя. Для более гибкого использования существуют флаги список которых можно просмотреть введя `ps --help all`.

> [!IMPORTANT]
> Наиболее часто используемые флаги которые могут вам встретяться - `ps aux` и `ps auxf`.  
> **a** (all) - Снимок процессов от всех пользоватeлей, а не только текущего    
> **u** (user-oriented) - Выводит дополнительную информацию о процессе - его PID, владельца, занимаемую память  
> **x** (extended) - Добавляет процессы не привязанные к конкретному терминалу. К примеру фоновые службы и демоны  
> **f** (forest) - Выводит информацию в виде древовидной структуры

## Сети
### `ping`
ping - простейшная утилита для работы с адресами. Проверяет жив ли сервер отправляя к нему тествовые пакеты данных.
![[Pasted image 20260405122037.png|438]]

**64 bytes from 8.8.8.8** - сервер (в нашем случае google.com) получил от нас запрос и отправил встречный пакет размера 64 байта. 
**icmp_seq** - порядковый номер пакета, используется для проверки "не потерялся ли пакет?"
**ttl** (Time To Live) - количество прыжков которое может совершить пакет от роутера к роутеру, с каждым прыжком от начального значения отнимается единица - когда значение будет равно нулю, то пакет уничтожится. Это нужно для того чтобы пакеты в случае неопределенного поведения не летали бесконечно по интернету
**time** - время за которое пакет прошел весь путь туда и обратно.
### `ss`
**ss** (Socket Statistics) - команда нужна для анализа сетевых соединений и отображения статистике сокетов. Показывает активные TCP, UDP, RAW, Unix-сокеты, какие порты октрыты и какие процессы их используют и является более свежей и лучшей заменой **netstat**
> [!TIP]
> Также полезная команда будет `ss -ntlp` - показывает список всех программ (и их PID) использующие порты.
> `-l` (listening) - только слушающие сокеты
> `-p` (process) - показывает процесс
### `curl`
**curl** (Client URL) - утилита для передачи данных по протоколам (HTTP / FPT и т.д.) используется для проверки доступности сайтов, API и скачивания файлов 

### `tcpdump`
**tcpdump** - мощная утилита для просмотра содержимого TCP пакетов. Может понадобится для отлеживания насколько безопасные пакеты доходят сайта, доходят ли они вообще, так как позволяет сортировать данные по конкретному IP-адресу, порту и сетевому интерфейсу. Если данные передаются по **http** то выводит содержимое пакетов

### `traceroute`
Утилита командной строки тесно связанная с **Time To Live** - параметр из команды **ping**. Но если пингуя сервер мы просто получаем сухой ответ от сервера, то traceroute с каждой итерацией кидает по 3 пакета (то есть 3 замера времени отправки-получения пакета) сначала с ttl = 1, получает ответ и выкидывает информацию - сколько времени ушло на отправку и получение пакета, откуда вообще пришел пакет (иногда показывает доменное имя отправителя), далее с ttl = 2, 3, 4 ... и таким образом мы увидим полный путь пакета от нас до нужного сервера. Звучит непонятно, но на примере станет яснее
![[Pasted image 20260405124616.png]]
Для примера будем использовать DNS от гугла. 
**Первый шаг** - мы отправили 3 пакета с TTL = 1, он дошел до нашего роутера (то есть был сделан один прыжок от нас до роутера) поэтому TTL - 1 = 0. Так как время жизни стало равно 0, наш роутер (В моем случае XiaoQiang - роутер от Xiaomi) отправил в ответ в виде `XiaoQiang (192.168.31.1)  4.299 ms  12.982 ms  12.976 ms`
`XiaoQiang` - сообственно доменное имя нашего роутера
`(192.168.31.1)` - IP адрес нашего роутера
`4.299 ms  12.982 ms  12.976 ms` - Так как мы отправили 3 пакета то и получили 3 замера с разным временем, нужно чтобы понять стабильность и разницу по времени нашей сети.
**Второй шаг** - теперь мы отправляем пакеты с TTL = 2, он снова дошел до нашего роутера и отнял единицу, TTL стал равен 1. Так как пакет ещё жив он идет дальше (выполняет следующий прыжок) и уже доходит до роутера нашего провайдера, там уже TTL = 0 поэтому пакет уничтожается и выплевывает статистику
Таким образом пакет будет прорывать себе путь пока не дойдет до нашей цели (В нашем случае до DNS гугла - 8.8.8.8)
> [!TIP]
> Полезные флаги команды **traceroute**
> `-I` (ICMP Echo) - по умолчению утилита шлет небезопасные UDP пакеты на специфичные порты, которые некоторые сайты часто закрывают. Данный флаг заставляет использовать TCP пакеты
> `-n` (No DNS) - выводит только IP адреса без доменов
> `-T` (TCP SYN) - Имитирует попытку открытия веб-сайта, требует **sudo**
> `-p <значание>` (Port) - Отправляет запрос на определенный порт
> `-q <значение>` (Queries) - По умолчанию отправляет по 3 пакета за каждый шаг, если вам не нужно знать время отклика можете установить `-q 1`
> `-w <время в секундах>` (Wait) - Устанавливает время ожидания ответа в секундах. Полезно когда вы отправяляйте запрос в другую страну или даже в другой конец планеты, когда времени ожидания по умолчанию может не хватить
