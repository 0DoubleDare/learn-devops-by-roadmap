# Git
База не только для DevOps, но и для каждого программиста. Лучший способ понять Git это создать свой репозиторий (было неплохо какой-нибудь реальный проект) и оттачивать навыки с контролем версий. Существуют разные облачные сервисы для хранения ваших репозиториев, наиболее популярный из них - **[Github](https://github.com/)**, но также советую не обделять такие альтернативы как **[Gitlab](https://gitlab.com/)** (Лучше для работы с CI/CD) и **[Bitbucket](https://bitbucket.org/product/)** (Интеграция с [Jira](https://ru.wikipedia.org/wiki/Jira) и [Trello](https://ru.wikipedia.org/wiki/Trello))
## **Основные определения**
**Репозиторий ( Repository )** - "хранилище", папка проекта храняща предыдущие файлы и историю изменений помимо текущих.

**Ветка ( Branch )** - Ссылка на конкретный коммит, так как они являются лишь ссылками то весят очень мало. Поэтому ветки нужно создавать часто особенно для совместной работы. Ветки можно: сливать, создавать, переключать.

**Тег ( Tag )** - Метка для ветки, отличия тега от ветки в том что теги никак нельзя перемещать и сливать (мержить). Их используют для версиорирования проекта (v0.1-alpha, v1.1-beta, v2.2-release).
